## 2. Deep Dives (Top 2)

### DeepSeek V4: Architectural Leap in Open-Weight Frontier Models

- **What happened:** DeepSeek released preview variants of V4 on April 24 — V4-Pro (1.6T total / 49B active MoE) and V4-Flash (284B total / 13B active). Both are open-weight, pre-trained on 32T+ tokens, and deploy 1M-token context as the default across all official DeepSeek services.
- **Technical details:**
  - **Hybrid Compressed Attention:** Combines Compressed Sparse Attention (CSA) for mid-range context with Heavily Compressed Attention (HCA) for distant tokens. HCA stores compact summary representations rather than full KV pairs, enabling 1M-token context at only 27% of V3.2's single-token inference FLOPs and 10% of its KV cache footprint.
  - **Manifold-Constrained Hyper-Connections (mHC):** Replace standard residual connections; strengthen signal propagation stability across deep layers while preserving model expressivity.
  - **Muon Optimizer:** Replaces AdamW for improved convergence speed and training stability at scale.
  - Both models are available on Hugging Face, NVIDIA NIM (Blackwell-optimized), and DeepInfra.
- **Why it matters:** First open-weight model credibly claiming near-parity with GPT-5.5 and Opus 4.7 on reasoning benchmarks. The 27% FLOPs figure at 1M context is a genuine architectural breakthrough — not a marketing claim. Commercial API pricing moat continues to erode.
- **Who should care:** AI infrastructure engineers, enterprises evaluating self-hosted LLM stacks, cost-sensitive agentic pipeline builders, anyone currently paying closed-API rates for reasoning tasks.

### Stryker Wiper Attack: MDM as a Weapon

- **What happened:** On March 11, 2026, Iranian threat group Handala (attributed to Iran's Ministry of Intelligence and Security, MOIS) used a compromised Microsoft Intune administrator account to issue mass remote-wipe commands against Stryker's global device fleet. Between 05:00–08:00 UTC, devices across 79 countries rebooted to blank screens showing the Handala logo. No conventional malware was deployed.
- **Technical details:**
  - **Initial access:** Infostealer malware harvested saved passwords, session tokens, and authentication cookies from an admin workstation.
  - **Privilege escalation:** Attacker created a new Global Administrator account in Microsoft Entra ID using the compromised credentials — no lateral movement required.
  - **Execution:** Microsoft Intune's legitimate "Wipe" MDM command issued at scale across the entire managed device fleet. Traditional EDR/AV solutions saw no malicious payload — they were bypassed entirely because the attack operated through an authorized management channel.
  - **Scale:** ~80,000 devices confirmed wiped; media reports citing up to 200,000 total affected devices across 79 countries.
  - **CISA response:** Issued advisory urging all organizations to harden Intune configurations and restrict Global Administrator role assignments.
- **Why it matters:** Establishes MDM infrastructure as a primary destructive attack vector. No malware means no signature-based detection. Any enterprise with unmonitored Intune admin accounts is exposed to this exact attack class.
- **Who should care:** Every enterprise using Microsoft Intune, Entra ID, or any MDM platform; CISOs; IR teams; healthcare and critical infrastructure operators.

## 5. Cybersecurity Report

### Incident 1: Stryker Corporation — MDM Wiper Attack
- **Target:** Stryker Corporation (global medical device manufacturer, $20B+ revenue)
- **Attack type:** Destructive wiper via MDM platform abuse — no malware deployed
- **Root cause:** Infostealer malware compromised Microsoft Intune Global Administrator credentials; insufficient Entra ID admin role restrictions and no approval workflow on bulk wipe commands
- **Attack strategy:** Credential harvest via infostealer → Entra ID Global Admin account creation → Mass Intune "Wipe" command issued across entire managed device fleet in a 3-hour window (05:00–08:00 UTC)
- **Impact:** 80,000–200,000 devices wiped across 79 countries; manufacturing, order processing, and global shipments disrupted; Q1 2026 earnings materially impacted; CISA emergency advisory issued; no patient data or connected medical devices affected

### Incident 2: Match Group — Vishing / Okta Credential Compromise
- **Target:** Match Group (Tinder, Hinge, OkCupid — 10M+ user records exposed)
- **Attack type:** Vishing (voice phishing) + Okta credential compromise → data exfiltration
- **Root cause:** Employees targeted via phone calls; MFA bypassed at the human layer rather than the technical layer
- **Attack strategy:** ShinyHunters group placed targeted calls impersonating IT helpdesk → employees surrendered Okta credentials → attacker authenticated and exfiltrated 10M+ records
- **Impact:** Major PII exposure across three major dating platforms; regulatory investigations ongoing; reputational fallout significant

### Incident 3: May 1, 2026 Ransomware Cluster
- **Target:** Multiple organizations including Abazia S.p.A., ATF Aerospace, Bimtrazer, Apotheca Beauty, Arban, and others
- **Attack type:** Ransomware (Qilin, INC_RANSOM, Akira, Nova, BlackNevas, ANUBIS)
- **Root cause:** Varied initial access vectors — likely unpatched VPN/RDP exposure, phishing-delivered loaders, and credential stuffing from infostealer databases
- **Attack strategy:** Multiple independent threat actors targeting different verticals (manufacturing, aerospace, healthcare, retail) simultaneously on the same date — potential coordinated "event day" timing across RaaS affiliates
- **Impact:** Operational disruptions and data exfiltration across manufacturing, aerospace, and healthcare sectors; scope still developing

### Patterns:
- **MDM/identity control plane is the new endpoint.** Attackers have fully pivoted from endpoint malware to weaponizing legitimate management infrastructure. Intune, Entra ID, and MDM platforms are now Tier-1 attack targets. Traditional EDR is blind to this class of attack.
- **Infostealer-as-initial-access is dominant** across all major 2026 breaches. Cheap, scalable, bypasses MFA by harvesting valid session tokens rather than cracking credentials.
- **Vishing resurges as MFA adoption grows.** As technical bypass becomes harder, attackers pivot to the human layer. Social engineering of helpdesk and IT staff is now a primary access vector for enterprise breaches.
- **RaaS fragmentation continues.** Six distinct ransomware groups active on a single date signals mature affiliate model infrastructure — barrier to entry for ransomware operations at near-zero for technical actors.

### Defensive Takeaways:
1. **Restrict Global Admin roles in Entra ID** — Implement time-bound Privileged Identity Management (PIM), require MFA on all admin accounts, and alert immediately on new Global Admin account creation.
2. **Gate MDM bulk actions** — Implement approval workflows before mass wipe commands execute; alert on any Intune wipe command issued at scale.
3. **Infostealer credential hygiene** — Deploy darkweb/paste-site credential monitoring with automated Okta/Entra session revocation; enforce short-lived tokens; mandate hardware FIDO2 for all privileged accounts.
4. **Vishing-resistant MFA** — Replace SMS/push MFA with phishing-resistant FIDO2 hardware keys for all SaaS identity providers (Okta, Entra ID). Train helpdesk on out-of-band verification for all credential reset requests.
5. **Isolate MDM from OT networks** — Ensure the device management plane cannot pivot to operational technology, manufacturing systems, or patient-facing infrastructure.
